# 招小助 · 成人学历提升电销 AI 副驾

这是一个本地运行的招生销售辅助工具，面向自学考试、成人高考、国家开放大学等咨询场景。它默认采用本地规则演示；配置模型接口后可生成更贴合上下文的草稿。

## 已包含的功能

- 学员画像：收集学历、目标、地区、时间、顾虑等必要信息。
- 方案建议：给出“可优先了解 / 可作为备选”的路径，不把建议伪装成资格结论。
- 销售话术：生成首聊、需求沟通、异议处理和下一步跟进草稿。
- 电销实时副驾：浏览器麦克风实时转写或粘贴通话转写后，识别费用、时间、考试、信任、用途等信号，并给出下一句和追问建议。
- 自动实时 AI：接入并启用真实 AI 后，可勾选“自动实时 AI”；每次新增的有效转写停顿约 2 秒后自动更新话术、追问、风险和画像。为控制调用量，短文本和高频片段会合并处理。
- 自动补充画像：仅从学员在通话中明确说出的非敏感信息中补齐空白字段；已手动填写的字段不会被覆盖。
- 电销查看界面：电销看板已前置，按“下一句 → 需核验/不要承诺 → 现在只差问”排列；可点顶部“专注查看”隐藏无关内容，通话时只保留重点。
- 通话纪要：通话后生成可复制到 CRM 的跟进摘要。
- 本机待跟进：保存于当前浏览器，可载入或导出 CSV；不会自动发消息、报名或收款。

## 已导入的两份内部资料

工具已安全整理了你提供的两份文件，规则保存在 [knowledge-base/organization-knowledge.json](knowledge-base/organization-knowledge.json)：

- 内部电销话术参考：只提取了开场、需求澄清、异议回应、约下一步的沟通结构；历史政策、资格、费用、时效和承诺性说法不会被引用。
- 内部班型目录：作为“待负责人复核”的项目目录。资料日期为 2026-05-12，且发现部分价格汇总不一致、一些页面含证书/截图样本，工具不会自动报价、承诺优惠、展示图片材料或把它们发给模型。

页面会明确显示这两个资料源的状态。要让班型、价格或招生条件能真正用于对外回复，请先补齐每条记录的适用省份/批次、官方来源、有效期、退款规则和复核人，再写入 `approvedFacts`。

## 启动

本项目没有第三方依赖，需要 Node.js 20+（当前环境已具备）。

```bash
npm start
```

打开终端提示的地址，通常是 <http://localhost:3000>。

开发时可使用：

```bash
npm run dev
```

## 公开预览与各自配置密钥

公开预览会部署到 GitHub Pages：<https://3977871701-spec.github.io/zhaoxiaozhu-sales-copilot/>。任何人都可以直接打开并体验本地规则、通话看板和浏览器本机保存功能。

公开预览不会携带任何人的模型密钥，也不会开放共享的 AI 接口。需要真实 MiniMax M3 时，访客可点击页面顶部的“配置 API Key”，选择中国区或国际区，输入**自己的** Key 并完成测试后使用真实 AI。

该临时 Key 只存在于当前页面内存：不会写入 localStorage、sessionStorage、Cookie、GitHub 或本项目服务器；刷新、关闭页面或重新打开页面都会清空。若需供自己的团队稳定使用，也可以自行克隆仓库，复制 `.env.example` 为 `.env`，填入自己的 Token Plan API Key 后运行 `npm start`；`.env` 已被 Git 忽略。

## 开启真实 AI

默认不用任何 API 就能试用本地模板。若要开启真实 AI：

1. 将 [.env.example](.env.example) 复制为 `.env`。
2. 填写 `OPENAI_API_KEY` 和你账号中可用的 `OPENAI_MODEL`；也可填写兼容 OpenAI Chat Completions 的 `OPENAI_BASE_URL`。
3. 重启服务，网页顶部显示“真实模型已连接”后，打开“使用真实 AI 生成”。

MiniMax M3 的配置值如下（不要把实际密钥提交到代码库）：

```env
# 国际区：https://api.minimax.io/v1；中国区：https://api.minimaxi.com/v1
OPENAI_BASE_URL=https://api.minimaxi.com/v1
OPENAI_MODEL=MiniMax-M3
```

启动时会校验 MiniMax 凭据是否能使用该模型。若顶部显示“模型密钥待更新”，页面会安全回退到本地规则模式；请在 MiniMax 控制台检查或重新生成已激活的 Token Plan API Key。

本机部署模式下，API 密钥只在本机服务端读取，不会写进浏览器代码。公开页的临时接入模式则由浏览器直接向使用者选择的 MiniMax 服务区域发起请求，Key 不会传给本项目服务器。工具会尝试隐藏常见敏感字段，但并非完全防漏；即使开了真实 AI，也不要粘贴身份证、银行卡、验证码、手机号、微信号或证件内容。

## 电销实时使用方式

1. 通话前先完成适用的录音/实时转写告知与授权，在页面勾选确认。
2. 建议使用桌面版 Chrome 或 Edge（ChatGPT 内嵌浏览器可能允许麦克风、但不允许 Web Speech 转写）。首次点击“开始麦克风转写”会检测或请求麦克风权限；成功后按提示再点击一次开始。也可以从电话系统粘贴已转写的文本。
3. 页面会即时给出本地提示，并自动补齐通话里明确说出的空白画像。配置真实 AI 后，可点“立即分析”，或勾选“自动实时 AI”，在每段有效转写停顿后自动更新建议和画像；手动填写的字段不被覆盖。
4. 通话结束点“生成通话纪要与跟进”，人工核对后再放进 CRM。

通话时可点击顶部“专注查看”，此时页面会隐藏画像、知识库和待跟进列表，把“下一句建议”放在最醒目的位置；再次点击或按 `Esc` 即可退出。

浏览器只能听到麦克风收到的声音。若希望稳定获取客户和销售双方的电话语音，应在第二阶段对接你们的云呼叫/座席/CTI 系统的合规录音流或实时转写 webhook。那一步需要知道你们使用的电话系统供应商、是否有 API/语音流权限、是否已完成录音告知；不建议从普通电话线路私自采集音频。

## 后续：接入你们自己的话术与知识库

请把你们现有的资料发来即可，优先顺序是：

1. 常用首聊、邀约、异议处理和跟进话术（Word、Excel、PDF、TXT 均可）；
2. 已审核的招生项目资料：省份、年度/批次、院校、专业、层次、条件、收费、退款规则、官方来源与更新时间；
3. 销售 SOP：什么时候追问、什么时候转主管/教务、什么话不能说；
4. 已脱敏的优秀电销录音转写或聊天记录。

收到后可将其整理为“已审核知识库”，每一条事实附来源、适用省份、有效期和复核人。知识库没有明确依据时，AI 会固定提示“需核验/转人工”，而不是猜测。修改 [organization-knowledge.json](knowledge-base/organization-knowledge.json) 后，重启本地服务即可加载新版本。

## 内置边界

- 不应输出或默认任何“包过、保录、免考、免试、内部名额、快速拿证、全日制、代考”等说法。
- 不对资格、录取、毕业、费用、退款、学历用途作未经核验的承诺。
- 不自动拨号、自动外呼、自动发消息、代替人工收款或提交报名。
- 面向学员的最终内容必须由销售人工审核；具体项目以当地教育考试主管部门、院校和已审核资料为准。
